# MJSDS Dashboard Charter

## Purpose

Maintain a lightweight business dashboard that can be safely deployed and updated from a simple static repository.

## Project Type

Published static dashboard.

## Current Stage

Active.

## Goal

Keep the live dashboard understandable, deployable, and low-friction to maintain.

## Core Question

Can this repository remain simple while still being reliable enough for repeated business use?

## In Scope

- dashboard UI and behavior
- setup and deployment documentation
- maintenance-safe updates

## Out Of Scope

- portfolio governance
- unrelated MJSDS website initiatives
- hidden secrets or environment-specific credentials in the repo

## Success Criteria

- the app remains deployable
- setup instructions still work
- update risk stays low

## Constraints

- OAuth and API setup are easy to misconfigure
- the project should stay simple rather than overbuilt

## Operating Principle

Every change should make the dashboard easier to trust or easier to maintain.
